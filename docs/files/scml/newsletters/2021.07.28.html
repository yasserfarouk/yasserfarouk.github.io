SCML Newsletter (July 28th 2021)

This is the sixth SCML newsletter.

SCML is one of the leagues of ANAC 2021 one of the official competitions of IJCAI 2021.

  1. Due to an error on our side, the remaining time displayed on the submission website was inaccurate for clients (depending on the time-zone). This was corrected and now the remaining time should be the same in all machines. To avoid a sudden shortening of the perceived remaining time for any participants, we extended the deadline by 12 hours to August 1st 11:59:59 AoE. Please be sure to submit your agent before this time as it is a hard deadline (i.e. the server will automatically stop receiving any new submissions exactly at this time).

  2. Other than the monetary prizes for the winner, second, and third place agents in the three tracks of SCML, we are introducing a new prize called “BIRD Innovation Prize” which can be given to “the most innovative strategy as measured by evaluation of the description (academic report) and implementation (code)”. An agent that was not in the top three according to the score can still win this prize which will be monetarily on-bar with the score-based prizes. Please be sure to make your academic report (and hopefully your code) as clear and descriptive as possible for a chance to win. It is possible to win this prize in addition to being one of the score-based winners in any SCML track.

  3. Our community is stabilizing:

    • 77 agents have been uploaded so far (18 in the last two weeks).
    • 33 teams are currently active in the online-competition (no change for two weeks).
    • No new registrants on the official website in the last two weeks. We are up to 53 registrants so far in 2021 (93 in total).

  4. Please be sure to check each of the following items before submitting your agent

    • Be sure to write the short description of your agent in the textarea added to the submission page. You can update it with the final report submission until August 10th.
    • If you are participating in the standard competition and do not have a collusion strategy, please uncheck “Run in collusion track.” in your submission page (Active Agent). According to the competition rules, if you do not have a non-trivial collusion strategy, the agent cannot advance to be a finalist on the collusion track anyway. By unchecking this box you allow us to run the competition at a lower cost with no loss for your agent.
    • Never use print(). You can save log information for debugging as described here.
    • Make sure that you do not use any plotting library in your agent (e.g. matplotlib, seaborn, plotly).
    • In SCML, agents are not allowed to pass information between world simulations. Never open a file for writing. You can read data in your agent’s path as described here. Trying to write to the server’s HDD, will lead to disqualification.
    • If you are using a data file in your agent, please DO NOT use pickle. The reason is that pickle is sensitive to the version number of all libraries involved and we cannot guarantee that our servers will have the same versions used to save the file. We cannot guarantee that your agent will behave normally if you are using pickle. Recommended storage formats are csv and json. For binary formats, check feather.

  5. Newer versions of NegMAS (0.8.7) and SCML (0.4.8) have been published. If you are using pip, upgrading is as simple as running pip install -U negmas scml. This time we mostly focused on bugfixes.

    • In some earlier versions, agents received catalog prices instead of trading prices from their AWIs. The system also used catalog prices instead of trading prices for setting negotiation agendas (not in accordance with the game description). This is now fixed and both agents and the system use trading prices. The main effect this will have on agents is that the negotiation agenda will be different every step (in accordance with the game description). A related issue caused bankrupt buyers to continue negotiating (and losing money) after bankruptcy. This is also solved.
    • We also added two new methods to the OneshotUFun class that you can access as self.ufun.ok_to_sell_at() and self.ufun.ok_to_buy_at(). These methods can be used to check if the price associated with an offer is totally crazy (i.e. it is impossible that buying/selling an item at this unit price can be beneficial). These two methods are provided as-is and are not guaranteed to work correctly in every case. You are free to use them if you want but we neither recommend nor require using them (You can get more information here). These methods are greedy in the sense that they do not consider any long term relationship issues with suppliers/customers or any other effects on the market other than the offer being evaluated (and only the unit price issue nothing more).
